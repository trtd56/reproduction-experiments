## 主要な発見

### 1. large-v3 + `language=ja` は無音・非音声の**100% (38/38)** で「ご視聴ありがとうございました」

条件（無音、ノイズフロア、白色/ピンク各レベル、ハム、純音）を問わず、出力は完全にこの一句に収束。
「モデルが大きいほどハルシネーションが消える」のではなく、**大きいほど『無音→定型句』の対応が強く・一貫して現れる**。

### 2. ハルシネーションの「型」はモデルサイズで質的に変わる（H4a 支持）

| モデル | ja での典型出力 | 型 |
|---|---|---|
| tiny | 「ご視聴ありがとうございました」(5x) / 数字列ループ | 定型句 + 反復 |
| base | 「スタッフを見つけたら…」×N | 反復ループ |
| small | 「んんんんん…」/ 音写列 | 退化した反復 |
| medium | 「あなたは…」ループ、「この動画をご視聴頂き申し訳ございません ご視聴頂きありがとうございました」 | 反復 + YouTube定型句の混成 |
| large-v3 | 「ご視聴ありがとうございました」(38/38) | 定型句に完全収束 |

小さいモデルは「流暢さの低い」反復・退化系列、大きいモデルは「流暢な」訓練データ定型句。
デコーダの言語モデルが強いほど、無音時の出力が高頻度字幕フレーズの attractor に落ちる、という説明と整合。

### 3. 定型句は言語トークンで切り替わり、**意味は言語をまたいで共通**（H2 強く支持）

同一音声に対し large-v3 は:
- `ja` → 「ご視聴ありがとうございました」
- `en` → "Thank you." / "Thank you for watching."
- auto → 上記に加え、ウェールズ語 "Diolch yn fawr"（=「ありがとうございました」）

**3言語とも「視聴への感謝」という同じ意味**の定型句に収束する。動画字幕の末尾定型
（無音区間に載る "thanks for watching" 系字幕）が言語ごとに学習されているという
訓練データ由来仮説の強い証拠。auto でウェールズ語が頻出するのは Whisper の既知の
言語誤判定バイアスとも一致。

なお、ユーザー例の「チャンネル登録ありがとうございます」は今回 0 件。
日本語の無音 attractor は「ご視聴ありがとうございました」（全体で74件）が支配的。

### 4. kotoba-whisper-v2.0 は**全条件 114/114 で「ごめん。」**（H4c は棄却方向）

蒸留でハルシネーションが消えるどころか、「無音→ごめん」という別の attractor に収束。
ただし注意: transformers pipeline 経由のため faster-whisper の no_speech/logprob 抑制が
効いていない。「蒸留カリキュラム（ReazonSpeech）の無音対応テキストが変わった」ことは
言えるが、抑制込みの実運用比較には同一デコードスタックでの再測定が必要。

### 5. 緩和策の効果は VAD ≫ 既定の抑制 > condition_on_previous_text（H5 支持）

tiny/base × 全10条件 × ja/auto での率:

| variant | 内容 | ハルシネーション率 |
|---|---|---|
| vad | Silero VAD 前段フィルタ | **0%（完全抑制）** |
| default | no_speech/logprob 抑制のみ | 条件により 60–100% |
| no_condition | 前文脈条件付けを切る | default とほぼ同等（一部条件で改善） |
| no_suppress | 抑制を全て無効化 | ほぼ 100% |

- 合成した定常ノイズに対しては VAD が完璧に機能する（実環境音・音楽ではすり抜けが予想される → ESC-50 で今後検証）。
- default と no_suppress の差 = 内蔵抑制が実際に落とせている分。ただし無音では
  `no_speech_prob≈0.94` かつ `avg_logprob≈-0.9` で「自信のある幻覚」が閾値
  （no_speech>0.6 **かつ** logprob<-1.0）をすり抜けるため、default でも高率で残る。

### 6. 「大きい音ほど危険」ではない。**最も危険なのは無音〜微小ノイズ**（H3 一部支持）

medium + ja では白色/ピンク -20dBFS（大きめのノイズ）で率 0% に落ちる一方、
純無音では 100%。音圧が上がるとエンコーダが「非音声」の証拠を得て抑制が効きやすく
なるが、無音はかえって定型句 prior を最大化する。ハム（50Hz）・純音（440Hz）のような
定常トーンはほぼ全モデルで 100% と、無音と同様に危険。

### 7. CTC 対照: デコーダ LM がなければ流暢な幻覚は**一切出ない**（H4b 支持、2026-07-02 追加）

wav2vec2 系 CTC モデル 2 種（`jonatasgrosman/wav2vec2-large-xlsr-53-japanese`,
`facebook/wav2vec2-base-960h`）に同じ合成非音声 38 ファイルを入力した結果、
**両モデルとも非空出力 0/38**。CTC はフレームごとの音響分類 + blank 縮約であり
自己回帰的な言語モデルを持たないため、音響エビデンスがなければ blank に潰れる。
「幻覚の発生源は encoder ではなく decoder の内部言語モデル」という説明の決定的な対照実験。
ログ: `ctc_20260702-141432.jsonl`

### 8. ESC-50 実環境音: VAD は人間由来の非音声（咳・笑い）で**すり抜ける**（2026-07-02 追加）

実環境音 6 カテゴリ × 5 クリップ（30 秒化、-6dBFS）× tiny/base/large-v3 × ja/auto:

| condition | default 率 | vad 率 | 備考 |
|---|---|---|---|
| rain | 0.5–0.6 | **0.0** | VAD 完勝 |
| keyboard_typing | 0.4–0.9 | **0.0** | VAD 完勝 |
| breathing | 0.8–0.9 | **0.0** | VAD 完勝 |
| clock_tick | 0.7–0.9 | 0.0–0.4 | large-v3 で一部すり抜け |
| coughing | 1.0 | **0.5–0.9** | 大幅すり抜け |
| laughing | 0.9–1.0 | **0.4–0.8** | 大幅すり抜け |

- Silero VAD は咳・笑い声を「音声」と判定して通してしまい、後段の Whisper が
  「ご視聴ありがとうございました」（large-v3 + ja で頻出）や擬音ループ
  （「はっはっはっ…」「wwww…」）を生成する。**VAD は定常環境音には完璧だが、
  人間の非言語音声には万能ではない**（会議録・医療文脈で問題になる Koenecke et al. の
  知見と接続する）。
- default 時の large-v3 + ja の出力には「ご視聴ありがとうございました」のほか
  「おやすみなさい」「音楽」「音量を調整します。」など、**字幕・キャプション由来と
  みられる別系統の定型句**も出現。訓練データ由来仮説をさらに補強。
- 図: `results/figures/esc50_vad_leak.png`。ログ: `esc50_20260702-141617.jsonl`

## 結論（説明ロジックの完成形）

1. Whisper のデコーダは字幕コーパスで学習された条件付き言語モデルであり、動画末尾の
   無音区間に載る定型字幕（ja:「ご視聴ありがとうございました」/ en: "Thank you for watching."）
   を「無音の書き起こし」として学習している（訓練データ由来）。
2. 無音・定常音では音響エビデンスが乏しく、出力は言語トークンに応じた定型句 attractor に
   収束する。モデルが大きいほど attractor が明瞭（large-v3 ja で 100%）。
3. この幻覚は「自信を持って」生成される（logprob が高い）ため、no_speech 検出と logprob の
   AND 条件である内蔵抑制をすり抜ける。パラメータ調整で消えにくいのはこのため。
4. 幻覚の発生源はデコーダの内部言語モデルである。CTC（デコーダ LM なし）は同一入力で
   非空出力 0/76 であり、encoder 側の問題ではないことが対照実験で確認できた。
5. 実効的な対策は音響ドメインでの VAD 前段フィルタ（合成定常音・雨・タイピング等では
   100% 抑制）。ただし咳・笑いなど人間由来の非言語音声は VAD を通過して幻覚が残る
   （すり抜け率 40–90%）ため、VAD + 既知定型句フィルタ等の多段防御が現実解。
