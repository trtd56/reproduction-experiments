## 1. 現象と通説、なぜ調べたか 2:00〜5:00

### ねらい

listener の「見たことある」を回収しつつ、通説と証拠のギャップを立てる。

### 話す内容

- X や Togetter で定期的にバズる現象。英語圏だと "Thank you for watching." が出る
- 通説:「YouTuber が動画末尾の無音部分に感謝字幕を入れる → Whisper は字幕と音声のペアで弱教師あり学習 → 無音＝この字幕、と学習された」
- 理屈はなんとなく分かる。でも「なんとなく」で止まっていた
- 疑問は4つ: 本当に訓練データ由来か / モデルでどう変わるか / 環境音で変わるか / なぜ設定をいじっても消えないのか
- アンカー論文は Barański et al. 2025（非音声入力による Whisper 幻覚の体系調査）
- FAccT 2024 の Koenecke らは「Whisper API の書き起こしの約1%に捏造フレーズ、38%は有害内容」「無音ポーズが長い失語症話者ほど幻覚が増える」と報告。笑い話では済まない文脈もある

### ひとこと

```text
「学習データのせいでしょ」って言うのは簡単なんですけど、
それを自分の手元の実験で示せるか、っていうのが今回のテーマです。
```

## 2. 実験設計 5:00〜8:00

### ねらい

規模感（1450回）と、3系統のモデルを並べた意図を伝える。

### 話す内容

- 音声は全部合成: 完全無音、マイクのノイズフロア相当の微小雑音、白色・ピンク雑音を3段階の音量で、50Hzのハム音、440Hzの純音。全部30秒
- あとから実環境音として ESC-50 データセットの雨・キーボード・時計の秒針・咳・呼吸・笑い声も追加
- モデルは3系統:
  - Whisper 5サイズ（tiny〜large-v3）→ サイズでどう変わるか
  - 日本語特化の蒸留モデル kotoba-whisper → 蒸留で消えるか
  - wav2vec2 系の CTC モデル → デコーダに言語モデルを持たない対照実験
- 言語設定（日本語・英語・自動判定）と緩和策（VAD など）もスイープ
- 全部 Mac の CPU でスクリプト一発。faster-whisper 様々

### 音声デモ案

- 「ノイズフロアってこういう音です」で -80dBFS の微小ノイズを流す（ほぼ聞こえない、それがミソ）

### ひとこと

```text
ポイントは、モデルを3系統並べたことです。
特に「デコーダに言語モデルを持たないモデル」を対照に置いたのが、あとで効いてきます。
```

## 3. 結果1: large-v3は無音の100%で「ご視聴ありがとうございました」 8:00〜12:00

### ねらい

一番インパクトのある数字を最初に出し、「大きいモデルほど安全」という直感を裏切る。

### 話す内容

- large-v3 に日本語設定で非音声を渡すと、38試行中38回、全部「ご視聴ありがとうございました」
- 無音でも、雑音でも、ハム音でも、全部この一句。むしろ大きいモデルほど現象が「純粋」になる
- ただし幻覚の「型」はサイズで変わる:
  - tiny: 定型句 or 数字の羅列ループ
  - base: 「スタッフを見つけたらスタッフを見つけたら…」の反復ループ
  - small: 「んんんんんん…」
  - medium: 反復と定型句の混成。「この動画をご視聴頂き申し訳ございません ご視聴頂きありがとうございました」という怪しい合成文も
  - large-v3: 「ご視聴ありがとうございました」一択
- 解釈: デコーダの言語モデルが強いほど、無音時の出力が「訓練データの高頻度フレーズ」という引き込み先（attractor）にきれいに落ちる

### 音声デモ案（この回のハイライト候補）

- 無音を3秒流す → 「これをlarge-v3に渡すと」→ 自分の声で「ご視聴ありがとうございました」→「100%です。38回中38回」

### ひとこと

```text
賢いモデルほど幻覚しない、じゃないんです。
賢いモデルほど、流暢に、自信を持って、同じ幻覚をするんです。
```

## 4. 結果2: 言語を変えると定型句が変わる。ただし意味は同じ 12:00〜15:00

### ねらい

訓練データ由来説の一番きれいな証拠を、聞いて分かる形で出す。

### 話す内容

- まったく同じ無音ファイルに対して、言語設定だけ変える
  - 日本語 → 「ご視聴ありがとうございました」
  - 英語 → "Thank you." "Thank you for watching."
  - 自動判定 → なぜかウェールズ語判定になって "Diolch yn fawr"（=どうもありがとう）
- 3言語とも「視聴への感謝」という同じ意味に収束する
- 動画末尾の感謝字幕は世界中の言語にある。それが言語ごとに学習されている、という説明そのもの
- 小ネタ: 有名な「チャンネル登録ありがとうございます」は1450回で0件。日本語の無音の主は「ご視聴〜」だった
- ほかに「おやすみなさい」「音楽」「音量を調整します。」という字幕・キャプション臭の強い句も出た

### ひとこと

```text
音は同じなのに、言語トークンを変えるだけで出てくる定型句が変わる。
これ、音を聞き取ってるんじゃなくて、「その言語の動画の終わり方」を思い出してるんですよね。
```

## 5. 結果3: 言語モデルを持たないモデルは、一切幻覚しない 15:00〜17:30

### ねらい

対照実験の気持ちよさを伝える。「犯人はデコーダ」の確定シーン。

### 話す内容

- CTC モデル（wav2vec2 系）はフレームごとに音を文字に分類するだけで、「次の単語を予測する」言語モデルを持たない
- 同じ非音声76ファイルを入れた結果: 非空出力ゼロ。全部空文字
- Whisper が60〜100%幻覚した同じ入力で、0%
- つまり「音を聞き間違えている（耳=エンコーダの問題）」ではなく「音がないときに口=デコーダが作文している」
- Calm-Whisper（Interspeech 2025）はデコーダ末尾層の少数の attention head が非音声幻覚の大半を駆動していると特定していて、これとも整合

### ひとこと

```text
しゃべる能力がないモデルは、幻覚もできない。
幻覚って、認識の失敗じゃなくて、生成の暴走なんです。
```

## 6. 結果4: なぜ設定をいじっても消えないのか 17:30〜20:30

### ねらい

一番技術的に面白い部分。「自信のある幻覚」という概念を持ち帰ってもらう。

### 話す内容

- Whisper には無音対策の抑制ロジックが入っている。セグメントを捨てるのは「無音確率が高い」かつ「生成の確信度が低い」の AND 条件
- 無音時の内部数値を見ると:
  - 無音確率 no_speech_prob は 0.94。モデルは無音だとほぼ気づいている
  - なのに生成テキストの対数確率 avg_logprob は -0.9 で、確信度が高い
- つまり「無音だと分かった上で、自信を持って幻覚する」。AND 条件の片方が成立しないので素通り
- 定型句は訓練データの高頻度系列だから生成確率が高い。幻覚が定型句である事実そのものが、抑制を無効化している
- 実測: 抑制を全部切るとほぼ100%、既定設定で60〜100%、condition_on_previous_text を切ってもほぼ変わらず
- アンカー論文の「パラメータ調整の効果は限定的」の理由がこの構造

### ひとこと

```text
「本当は無音だと分かってるのに、自信満々に嘘をつく」。
この2つの数字を並べて見たときが、今回の実験で一番ゾクッとした瞬間です。
```

## 7. 結果5: VADは効く。ただし咳と笑い声で破れる 20:30〜24:00

### ねらい

実務の takeaway。「VAD入れたから大丈夫」の落とし穴を伝える。

### 話す内容

- 唯一ちゃんと効いたのは VAD（音声区間検出）の前段フィルタ。合成音では全条件0%
- ところが実環境音だと: 雨・キーボード・呼吸は0%のまま、咳は50〜90%、笑い声は40〜80%がVADを通過
- VADが咳や笑いを「人の声」と判定して通してしまい、その先で Whisper が「ご視聴ありがとうございました」や「はっはっはっ…」「wwww」を生成
- 実務への示唆: 咳・笑い・相槌の多い音声、つまり会議録や診察録こそ危険
- Koenecke らの「失語症話者（発話間ポーズが長い）で幻覚率が上がる」報告と同じ構図。アクセシビリティの問題でもある
- 現実解は多段防御: VAD + 既知定型句フィルタ + 無音確率の単独利用（AND条件にしない）

### 音声デモ案

- ESC-50 の咳クリップを2秒流す →「これ、VADは『人の声』って言うんですよ。まあ、人の声ではあるんですけど」

### ひとこと

```text
「VADを入れたから安心」ではなくて、
咳と笑い声の多い会議室が、いちばん危ない場所なんです。
```

## 8. おまけとクロージング 24:00〜27:00

### ねらい

kotoba-whisper の小ネタで笑いを作ってから、まとめて記事に誘導。

### 話す内容

- おまけ: 日本語特化の蒸留モデル kotoba-whisper に同じ非音声を渡すと、114試行全部「ごめん。」
  - 蒸留で幻覚は消えず、引き込み先が別の句に置き換わっただけ（おそらく蒸留データのReazonSpeech由来）
  - 無音を渡すとひたすら謝ってくる文字起こしモデル
- まとめの5段ロジックを口頭で:
  1. 訓練字幕由来の定型句がある（言語を変えると意味が同じ句に切り替わるのが証拠）
  2. 無音ではデコーダの言語モデルの引き込み先に落ちる（大きいモデルほど純粋）
  3. 幻覚は自信を持って生成されるので内蔵抑制のAND条件をすり抜ける
  4. 発生源はデコーダ（言語モデルなしのCTCは幻覚ゼロ）
  5. VADは定常音に完璧、でも咳・笑いで破れるので多段防御が現実解
- 制約の正直な開示: 合成音中心、music や babble は未検証、シード5回

### 締め案

```text
「無音のとき、モデルは何も聞いていない。思い出しているだけ」。
今回はそれを、1450回の実験と対照実験で確かめた回でした。

ヒートマップとか、実際の幻覚出力の一覧は記事にまとめています。
概要欄のリンクから見てみてください。

あと、この番組の最後に僕が「ご視聴ありがとうございました」って言うと
紛らわしいので、今日は別の挨拶にします。
それでは、また次回。おつかれさまでした。
```

論文「Investigation of Whisper ASR Hallucinations Induced by Non-Speech Audio」をアンカーに、モデル5サイズ×ノイズ条件×緩和策の1450試行と、言語モデルを持たないCTCモデルとの対照実験を行いました。

## 収録メモ

- **音声デモがこの回の武器**。無音・微小ノイズ・咳のクリップは `experiment/data/processed/audio/` にある。編集時に挟む位置は本文の「音声デモ案」参照。冒頭の「無音を流す」は放送事故っぽくなるので2〜3秒で切ってすぐツッコむ
- 数値の読み方: no_speech_prob は「無音確率」、avg_logprob は「生成の確信度」と言い換えて、英語名は各1回だけ添える。dBFS は「音量レベル」で通す
- モデル名の読み: large-v3「ラージブイスリー」、CTC「シーティーシー」、VAD「バド」ではなく「ブイエーディー」、kotoba-whisper「コトバウィスパー」、ESC-50「イーエスシーフィフティ」
- ウェールズ語 "Diolch yn fawr" は無理に正確に読まず「ディオルヒ・ン・ヴァウル、みたいな発音らしいです」と逃げる
- 「1%が捏造・38%が有害」（Koenecke）は深刻な話なので、笑いのトーンから一段落として話す
- 締めの挨拶は台本どおり「ご視聴ありがとうございました」を封印するのを忘れない（ここが今回のオチ）
