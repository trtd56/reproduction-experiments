# Publishing Checklist

- [x] Public code is separated from the private study workspace.
- [x] No generated audio, local model cache, `.env`, API key, or private note is included.
- [x] LFM and Gemini variants are represented as config files.
- [x] Public README explains setup, run commands, and constraints.
- [x] Public docs include qualitative findings and variant descriptions.
- [x] Tests pass with `PYTHONPATH=src python -m pytest tests -q`.
- [ ] Add the final blog URL after publication.
- [ ] Decide the public repository license.
- [ ] Re-check Gemini Live preview model names before posting if publication is delayed.
